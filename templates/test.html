Shared Dictionary Compression Test

How This Test Works

Step 1: This page loads /dictionary.js as a script tag. Chrome sees the Use-As-Dictionary header and caches the dictionary.
Step 2: After a brief delay, the page loads /bundle.js as a script tag. If Chrome has registered the dictionary, it will send Available-Dictionary and Accept-Encoding: dcb, dcz headers.
Step 3: Check DevTools Network tab for the /bundle.js request to verify dictionary compression is working.

What to Look For in DevTools

Open DevTools > Network tab and look at the /bundle.js request:

  • Request Headers:
    • Accept-Encoding should include dcb and/or dcz
    • Available-Dictionary should contain a base64 hash
    • Dictionary-ID should be "dict-v1"
  • Response Headers:
    • Content-Encoding should be dcb or dcz
    • X-Compression-Type shows what compression was used
    • X-Dictionary-Full-Match should be true

Troubleshooting

  • No Available-Dictionary header?
    • Make sure "Disable cache" is OFF in DevTools
    • Try refreshing the page (Cmd+R or F5) - the dictionary needs to be cached first
    • Chrome needs HTTPS (which we have via cloudflared tunnel)
  • Still not working?
    • Clear site data: DevTools > Application > Storage > Clear site data
    • Close and reopen the tab
    • Visit this page again and refresh once
